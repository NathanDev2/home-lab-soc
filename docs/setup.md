## Objective

The goal of this project is to build a basic home Security Operations Center (SOC) lab using an Ubuntu Server and Wazuh. This lab is designed to practice Linux server administration, endpoint monitoring, log collection, alert investigation, and basic incident response workflows in a controlled environment.

## Lab Overview

```text
Windows Endpoint
└── Wazuh Agent
    └── Sends endpoint security events

        ↓

Ubuntu Server
├── Wazuh Manager
├── Wazuh Indexer
└── Wazuh Dashboard
```

## Server Hardware

The Wazuh server was installed on a spare desktop PC.

| Component | Specification |
|---|---|
| CPU | Intel Core i5-9600K |
| RAM | 16 GB DDR4 |
| Storage | SSD storage |
| Operating System | Ubuntu Server |
| Role | Wazuh server / SOC platform |

## Step 1: Install Ubuntu Server

Ubuntu Server was installed on the spare PC to act as the central server for the SOC lab.

A bootable USB installer was created using balenaEtcher. The ISO file must be written to the USB as a bootable image; simply dragging the `.iso` file onto the USB drive does not create a bootable installer because the USB needs the proper boot structure.

### Installation Notes

During Ubuntu Server installation:

- Ubuntu Server was installed on the spare PC.
- OpenSSH Server was enabled.
- A normal user account was created.
- The server was assigned a recognizable hostname.
- The system was connected to the local network.

## Step 2: Enable Remote Administration with SSH

After installation, SSH was used to remotely manage the Ubuntu server from the Windows workstation.

SSH stands for Secure Shell. It allows a user to securely access a command-line session on another machine over the network.

Example SSH command:

```powershell
ssh username@SERVER_IP
```

For public documentation, the actual username and IP address are redacted.

## Step 3: Update the Ubuntu Server

After the first login, the server packages were updated:

```bash
sudo apt update
sudo apt upgrade -y
```

Basic administration tools were installed:

```bash
sudo apt install -y curl wget git vim htop net-tools unzip ufw
```

## Step 4: Configure the Firewall

UFW was enabled to provide a basic host firewall on the Ubuntu server.

SSH was allowed before enabling the firewall to avoid locking out remote access:

```bash
sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status verbose
```

Expected result:

```text
Status: active
OpenSSH ALLOW IN Anywhere
```

## Step 5: Install Wazuh

Wazuh was installed using the all-in-one installation assistant. This installs the Wazuh manager, indexer, and dashboard on the Ubuntu server.

```bash
curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh
sudo bash ./wazuh-install.sh -a
```

After installation, the Wazuh dashboard credentials were generated by the installer.

To retrieve the generated passwords:

```bash
sudo tar -O -xvf wazuh-install-files.tar wazuh-install-files/wazuh-passwords.txt
```

> Do not upload `wazuh-passwords.txt`, screenshots of passwords, or any credentials to GitHub.

## Step 6: Access the Wazuh Dashboard

The Wazuh dashboard was accessed from the Windows workstation using a browser:

```text
https://SERVER_IP
```

The dashboard uses HTTPS on port `443/tcp`. A browser certificate warning is expected in a home lab because the dashboard uses a self-signed certificate by default.

The firewall was configured to allow HTTPS access:

```bash
sudo ufw allow 443/tcp
sudo ufw status verbose
```

## Step 7: Verify Wazuh Services

The following services were verified on the Ubuntu server:

```bash
systemctl is-active wazuh-manager wazuh-indexer wazuh-dashboard
```

Expected output:

```text
active
active
active
```
